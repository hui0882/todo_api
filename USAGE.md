## 快速启动

### 方式一：一键启动（推荐）

```bash
./start.sh
```

这个脚本会自动启动后端和前端服务。

### 方式二：分别启动

**启动后端：**
```bash
# 使用 Go 直接运行
go run main.go

# 或使用 Makefile
make run
```

**启动前端：**
```bash
cd frontend
npm install  # 首次需要安装依赖
npm run dev
```

## 访问地址

- **后端 API**: http://localhost:8080
- **前端界面**: http://localhost:3000
- **健康检查**: http://localhost:8080/health

## 功能演示

### 1. 用户注册和登录

1. 访问 http://localhost:3000
2. 点击"注册"标签
3. 输入用户名（至少3个字符）、密码（至少6个字符）
4. 可选填写邮箱
5. 点击"注册"按钮
6. 注册成功后切换到"登录"标签
7. 输入刚才注册的用户名和密码
8. 点击"登录"按钮

### 2. 管理待办事项

登录成功后会自动跳转到待办列表页面：

**创建待办：**
1. 点击右上角"新建"按钮
2. 输入标题（必填）
3. 输入描述（可选）
4. 点击"创建"

**编辑待办：**
1. 在列表中找到要编辑的待办
2. 点击"编辑"按钮
3. 修改内容
4. 点击"更新"

**标记完成：**
1. 勾选待办前的复选框
2. 待办会显示删除线和"已完成"标签

**删除待办：**
1. 点击"删除"按钮
2. 确认删除

### 3. 配置 API 接口

**重要提示：API 配置功能无需登录即可使用，可以在登录前配置好后端地址。**

如果后端地址不是默认的 `http://localhost:8080`：

**方式一：登录前配置（推荐）**
1. 在登录页面，点击右上角"API 配置"按钮
2. 修改"服务器地址"
3. 根据实际情况修改各接口路径
4. 点击"测试连接"验证配置
5. 点击"保存配置"
6. 点击"去登录"返回登录页面

**方式二：登录后配置**
1. 登录后点击右上角"设置"按钮
2. 修改配置并保存
3. 返回首页继续使用

### 4. 使用压测工具

1. 点击右上角"压测"按钮
2. 配置压测参数：
   - **服务器地址**: 目标服务器（默认 localhost:8080）
   - **请求方法**: 选择 GET/POST/PUT/DELETE
   - **接口路径**: 如 `/health` 或 `/api/v1/todos`
   - **请求体**: POST/PUT 请求需要填写 JSON 数据
   - **总请求数**: 压测总次数（建议100-1000）
   - **并发数**: 同时发送的请求数（建议10-50）
3. 可选：点击"快速场景"按钮加载预设配置
4. 点击"开始压测"
5. 查看实时监控图表和统计结果

**压测结果说明：**
- **平均响应时间**: 所有请求的平均耗时
- **P50**: 50%的请求在此时间内完成
- **P90**: 90%的请求在此时间内完成
- **P95**: 95%的请求在此时间内完成
- **P99**: 99%的请求在此时间内完成
- **成功率**: 返回 2xx 状态码的请求占比

## 常见使用场景

### 场景1：验证后端接口是否正常

1. 进入压测页面
2. 服务器地址填写你的后端地址
3. 接口路径填写 `/health`
4. 请求方法选择 GET
5. 总请求数设为 10，并发数设为 1
6. 开始压测
7. 查看成功率是否为 100%

### 场景2：测试登录接口性能

1. 进入压测页面
2. 点击"登录接口"快速场景
3. 修改请求体中的用户名和密码为实际存在的账号
4. 设置总请求数为 100，并发数为 10
5. 开始压测
6. 观察平均响应时间和成功率

### 场景3：压测待办列表查询性能

1. 确保已登录（因为需要 Session Cookie）
2. 进入压测页面
3. 点击"获取TODO列表"快速场景
4. 设置总请求数为 500，并发数为 20
5. 开始压测
6. 查看 P95 和 P99 响应时间

### 场景4：更换后端地址

如果后端部署在其他服务器：

**推荐：登录前配置**
1. 在登录页面点击"API 配置"按钮
2. 将服务器地址改为 `http://your-server-ip:8080`
3. 点击"测试连接"
4. 测试通过后点击"保存配置"
5. 点击"去登录"返回登录

**或者：登录后配置**

1. 进入设置页面
2. 将服务器地址改为 `http://your-server-ip:8080`
3. 点击"测试连接"
4. 测试通过后点击"保存配置"
5. 返回待办列表，所有请求会自动使用新地址

## 故障排查

### 问题1：前端无法连接后端

**症状：** 所有请求都返回网络错误

**解决方法：**
1. 检查后端是否启动：访问 http://localhost:8080/health
2. 检查前端配置：进入设置页面，点击"测试连接"
3. 检查浏览器控制台的错误信息
4. 确认后端没有 CORS 错误

### 问题2：登录后立即被退出

**症状：** 登录成功后刷新页面需要重新登录

**解决方法：**
1. 检查浏览器是否阻止了 Cookie
2. 检查后端 Session 配置
3. 查看浏览器控制台是否有 Cookie 相关错误

### 问题3：压测结果全部失败

**症状：** 压测成功率为 0%

**解决方法：**
1. 检查请求地址是否正确
2. 检查请求方法是否正确
3. POST/PUT 请求检查请求体格式
4. 需要登录的接口要先登录
5. 查看浏览器控制台的详细错误

### 问题4：待办列表为空

**症状：** 登录后看不到任何待办

**解决方法：**
1. 这是正常的，新用户没有待办事项
2. 点击"新建"按钮创建第一个待办
3. 如果后端有数据但前端显示为空，检查 API 配置

### 问题5：前端依赖安装失败

**症状：** npm install 报错

**解决方法：**
```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 重新安装
cd frontend
rm -rf node_modules package-lock.json
npm install

# 或使用 yarn
yarn install
```

### 问题6：后端启动失败

**症状：** go run main.go 报错

**可能原因和解决方法：**

1. **MySQL 连接失败**
   - 检查 MySQL 是否启动
   - 检查 config.yaml 中的数据库配置
   - 确认数据库 todo_list_api 已创建

2. **Redis 连接失败**
   - 检查 Redis 是否启动
   - 检查 config.yaml 中的 Redis 配置

3. **端口被占用**
   ```bash
   # 查看占用端口的进程
   lsof -i :8080
   # 杀死进程
   kill -9 <PID>
   ```

4. **Go 依赖问题**
   ```bash
   go mod tidy
   go mod download
   ```

## 性能建议

### 压测建议

1. **本地测试**：总请求数 100-500，并发数 10-20
2. **开发环境**：总请求数 1000-5000，并发数 20-50
3. **生产环境**：请使用专业工具（JMeter、wrk、Apache Bench）

### 浏览器限制

- 浏览器对同一域名的并发连接数有限制（通常 6-8 个）
- 大并发压测建议使用服务器端工具
- 本工具主要用于快速验证和简单压测

1. **后端日志**: 查看 `logs/` 目录下的日志文件

3. **API 调试**: 使用 Postman 或 curl 测试接口
4. **数据库调试**: 直接查询 MySQL 数据库
